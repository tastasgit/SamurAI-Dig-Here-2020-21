SamurAI Dig Here 2020 Game Manager

IPSJ Programming Contest Committee
2021/01/05

The SamurAI Dig Here 2020 game manager is a tool to play games of SamurAI Dig Here 2020 between player agent programs and generates log of the games.

Functionalities

The SamurAI Dig Here 2020 game manager is for the following.

Playing games:
Given a game field configuration, a game by between two player player agent programs can be played, writing out the play log of the game.
Generating communication logs:
During games, copies of the data sent from the manager to player agent programs can be saved to an external file, which may be useful for debugging player agent programs.
Verifying play logs:
A play log file given can be checked for its consistency.

Usage

The command line should be in the following format. 

./manager [options] <log> [<script1> <script2> [<script3> <script4>]]

Game Field

The argument <log> is a path to a file describing the game field configuration and (possibly empty) play logs on the field. This file is usually prepared with the SamurAI Dig Here 2020 web page with the file name extension ".dighere".

Player Programs

When all of the arguments <script1> through <script4> are omitted, the manager checks the consistency of the log file.

When all of <script1> through <script4> are given, they are interpreted as paths to the shell script to launch player agent processes. <script1> and <script2> are for samurai while <script3> and <script4> are for dogs.

When only <script1> and <script2> are given, they are used also for launching processes for dogs. In this case, two distinct player agent processes of a team, one for samurai and another for a dog, are launched with the same script. Whether a process controls a samurai or a dog can be known by the information the manager sends to the player processes.

Options

The following options are available.

-S:
Output summary of each game step played to the standard error. The summary consists of the following lines. 
Step: 66
Agent 0@(0,5) move (0,6)  9330 msec left
Agent*1@(1,7) move (2,8)  7941 msec left
Agent 2@(1,8) move (2,7)X 9340 msec left; Crossing move lines
Agent 3@(0,4) move (1,5)  7785 msec left
Scores are 223:103
  • The step number of the game step.
  • Information on apparent treasures: their amounts and positions.
  • The positions, play plans, and think time left of four agents, one line for each agent. The agent number is prefixed with an asterisk * if the agent had taken a rest in the previous step and thus plays to diagonal directions were valid. The coordinates of cell planned to move to, dig up, or fill up are also displayed. When the plan cannot be actually taken for the plan was invalid or not viable, it is followed by an X to indicate it, and the point of inappropriateness of the plan is explained at the end of the line.
  • Scores of both teams up to this step.

-D <path>
Copies of the data send from the manager to player programs are saved in to the files with specified path. As there are four agents, the output goes to four files with paths generated by appending agent IDs to the <path> specified. The IDs are the following.
0:Samurai of player 1
1:Samurai of player 2
2:Dog of player 1
3:Dog of player 2

Player Processes

Player programs are run as children processes of the manager. Two processes each for both teams, one controlling the samurai and the other controlling the dog. In the tournament, they run the same program. Which agent is to controlled can be known by the data sent from the manager.

Each of the processes is given game status data from the standard input stream in each game step. It should respond with a play plan for the step. Further details can be found in the document SamurAI Dig Here 2020 Game Rules.
